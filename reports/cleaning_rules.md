# clean_rules.md

## 文件说明
本文件用于规范本项目在文献导出、字段统一、数据清洗、初步筛选与去重过程中的处理规则。  
本文件服务于以下目标：

1. 保证不同数据库数据在进入正式分析前具备统一结构；
2. 保证每一步清洗操作可追溯、可解释；
3. 为后续 `data_quality.md`、`processed`、知识图谱分析和论文写作提供统一依据。

当前项目已实际涉及两类数据源：
- 中国知网（CNKI）
- Web of Science（WoS）

由于两类数据源字段体系不同、检索噪声不同、题录质量不同，因此本文件采用“先分库清洗，再统一映射”的原则。

---

## 一、总原则

### 1. 分库处理，不直接混表
CNKI 与 WoS 原始数据不得直接合并后再清洗。  
必须先分别完成：
- 字段识别
- 基础去噪
- 文本规范化
- 初步筛选
- 唯一标识保留

之后才能进入统一字段映射与合并阶段。

### 2. 保留原始数据
所有清洗操作都必须建立在“保留原始文件”的前提下进行。  
原始 TXT、CSV、题录导出文件不得覆盖。  
清洗后的数据应另存为新文件。

### 3. 优先保留核心字段
无论数据库来源如何，以下字段优先保留：
- 题名
- 作者
- 单位 / 机构
- 摘要
- 年份
- DOI

如数据库支持，建议额外保留：
- 文献来源
- 文献类型
- 基金
- 参考文献
- 被引次数
- 唯一标识

### 4. 先清洗格式，再做相关性判断
不要在字段还混乱、文本还没统一前就做正式筛选。  
正确顺序是：
1. 统一字段
2. 清除明显错误值
3. 标准化文本
4. 去重
5. 初筛相关性

---

## 二、数据源分项清洗规则

## 2.1 CNKI 数据清洗规则

### 2.1.1 当前主要字段
CNKI 当前已出现或已稳定导出的字段主要包括：
- `SrcDatabase-来源库`
- `Title-题名`
- `Author-作者`
- `Organ-单位`
- `Source-文献来源`
- `Summary-摘要`
- `PubTime-发表时间`
- `Year-年`
- `Fund-基金`
- `DOI-DOI`

### 2.1.2 字段重命名规则
建议将 CNKI 原始字段统一映射为以下标准字段名：

- `SrcDatabase-来源库` → `source_type`
- `Title-题名` → `title`
- `Author-作者` → `authors`
- `Organ-单位` → `affiliations`
- `Source-文献来源` → `source`
- `Summary-摘要` → `abstract`
- `PubTime-发表时间` → `pub_time`
- `Year-年` → `year`
- `Fund-基金` → `fund`
- `DOI-DOI` → `doi`

### 2.1.3 文本标准化规则
1. 删除字段值首尾空格；
2. 统一中英文分号、逗号和空格；
3. 标题中的连续空格替换为单个空格；
4. DOI 统一转为小写或统一原样保存，但同一数据表中口径必须一致；
5. 作者字段统一使用分号 `;` 作为分隔符；
6. 单位字段保留原始机构串，不在初步清洗阶段强行拆分。

### 2.1.4 年份与时间规则
1. `PubTime` 保留原始日期；
2. `Year` 作为正式年度分析字段；
3. 若 `Year` 缺失但 `PubTime` 存在，可从 `PubTime` 中提取年份补全；
4. 若二者冲突，以 `Year` 为主，并在备注中登记异常。

### 2.1.5 来源库简写规则
为便于表格查看，可在清洗表中增加简写列：
- 外文期刊 → 英刊
- 期刊 → 期刊
- 硕士 → 硕论
- 博士 → 博论
- 会议 → 会议

原始值保留，不覆盖原字段。

## 2.2 WoS 数据清洗规则

### 2.2.1 当前主要字段
WoS 当前导出中典型字段包括：
- `PT`
- `AU`
- `AF`
- `TI`
- `SO`
- `DT`
- `AB`
- `C1`
- `C3`
- `RP`
- `EM`
- `FU`
- `FX`
- `CR`
- `TC`
- `Z9`
- `PY`
- `DI`
- `UT`
- `DA`

### 2.2.2 字段重命名规则
建议统一映射为：

- `PT` → `publication_type`
- `AU` → `authors`
- `AF` → `authors_full`
- `TI` → `title`
- `SO` → `source`
- `DT` → `document_type`
- `AB` → `abstract`
- `C1` → `affiliations`
- `C3` → `organizations`
- `RP` → `corresponding_author`
- `EM` → `email`
- `FU` → `fund`
- `FX` → `fund_text`
- `CR` → `references`
- `TC` → `cited_times`
- `Z9` → `total_cited_times`
- `PY` → `year`
- `DI` → `doi`
- `UT` → `unique_id`
- `DA` → `download_date`

### 2.2.3 WoS 特殊规则
1. `UT` 必须保留，作为 WoS 唯一标识；
2. `CR` 不得在初步清洗阶段删除；
3. `TC` 与 `Z9` 同时存在时，优先保留两者；
4. `AF` 比 `AU` 更完整，若后续只保留一个作者字段，优先保留 `AF`；
5. `C1` 与 `C3` 若同时保留，则 `C1` 用于详细机构信息，`C3` 用于标准机构归并。

## 三、统一字段映射规则

在 CNKI 与 WoS 分别清洗完成后，再进入统一字段阶段。  
建议统一保留以下字段：

- `database`
- `source_type`
- `title`
- `authors`
- `authors_full`
- `affiliations`
- `source`
- `document_type`
- `abstract`
- `year`
- `pub_time`
- `fund`
- `doi`
- `references`
- `cited_times`
- `unique_id`
- `language`
- `source_file`

### 3.1 必保字段
以下字段进入统一表时优先级最高：
- `title`
- `authors`
- `abstract`
- `year`
- `doi`

### 3.2 条件保留字段
以下字段有则保留，无则留空：
- `affiliations`
- `fund`
- `references`
- `cited_times`
- `unique_id`

### 3.3 允许为空字段
以下字段允许缺失，但不能乱填：
- `authors_full`
- `corresponding_author`
- `email`
- `source_type`

## 四、去重规则

### 4.1 去重优先级
去重按以下顺序执行：

1. `doi` 完全一致 → 判定为重复记录；
2. 若 DOI 缺失，则使用 `title + year` 联合判断；
3. 若标题仅有大小写、空格、连字符差异，视为同题记录；
4. 中英文题目明显不是同一篇时，不因关键词相似而合并。

### 4.2 去重保留原则
若同一文献在多个数据库同时出现，保留规则如下：

1. 优先保留字段更完整者；
2. 若 WoS 版本含 `references`、`cited_times`、`UT`，通常优先保留 WoS；
3. 若 CNKI 版本摘要更完整，可将摘要补充到统一表，但需在备注中记录数据来源；
4. 不删除原始来源信息，应保留 `database` 和 `source_file`。

### 4.3 不应误判为重复的情况
以下情况不视为重复：
- 同题不同年份的会议版与期刊版；
- 学位论文与后续改写期刊论文；
- 同一主题但不同作者、不同 DOI 的方法改进文献。

## 五、相关性筛选规则（清洗阶段初筛）

### 5.1 纳入标准
文献需尽量同时满足以下条件：

1. 主题核心属于电力负荷预测或电力需求预测；
2. 方法核心涉及 LSTM 或其明确变体；
3. 研究对象位于电力系统、电网、智能电网、配电网等电力场景中；
4. 能够为后续热点分析、方法演化分析提供有效证据。

### 5.2 排除标准
以下情况在清洗阶段可初步标记为排除候选：

1. 主题核心为微电网调度优化，而非负荷预测；
2. 主题核心为需求响应设计、建筑能耗管理、医院能源系统、船舶微电网等边缘场景，且负荷预测不是主任务；
3. 仅在摘要中附带出现 LSTM 或预测模块，但研究主体并非电力负荷预测；
4. 与交通流、网络流量、服务器负载、桥梁荷载等跨域主题相关；
5. 综述范围过大、无法明确归入本项目口径者。

### 5.3 边界文献处理规则
对于“有一定相关性，但未必应纳入最终样本”的文献，不直接删除，而是标记为：
- `keep = maybe`
- `exclude_reason = 边界文献`
- `note = 需二次人工判断`

## 六、建议新增字段

在清洗后的统一表中，建议新增以下人工处理字段：

- `keep`：是否保留（yes / no / maybe）
- `exclude_reason`：排除原因
- `note`：备注
- `screen_stage`：筛选阶段（title / abstract / fulltext）
- `database`：数据库来源
- `source_file`：原始文件名

这样便于后续：
- 生成 `processed`
- 编写 `data_quality.md`
- 回溯筛选过程
- 支撑 PRISMA 统计

## 七、文件保存规则

### 7.1 原始文件
原始导出文件统一放入：
- `data/raw/`

### 7.2 清洗中间表
字段重命名、文本标准化、初筛标记后的中间表建议放入：
- `data/interim/`

### 7.3 最终分析表
用于后续知识图谱分析、趋势图制作、统计分析的表建议放入：
- `data/processed/`

### 7.4 文档说明
本文件 `clean_rules.md` 属于规则说明文档，应放入：
- `reports/`
