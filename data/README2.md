# 基于 LSTM 的电力负荷预测研究热点与发展趋势——文献计量分析（2015–2025）

## 1. 项目概述
本项目为文献计量学课程项目，研究主题为“基于 LSTM 的电力负荷预测研究热点与发展趋势——文献计量分析（2015–2025）”。项目目标是围绕 2015–2025 年间相关文献，系统梳理 LSTM 及其改进模型在电力负荷预测领域中的应用现状、研究热点、主题演化、核心作者与机构分布，以及潜在的发展方向。

从研究对象看，项目聚焦于“方法—任务—场景”三者共同成立的文献：方法上限定为 LSTM 及其代表性变体；任务上聚焦电力负荷预测、电力需求预测、短期负荷预测等；场景上限定在电力系统、智能电网、电网、配电网等相关语境中。项目不把一般的时间序列预测、交通流预测、服务器负载预测等明显跨域文献作为正式研究对象。

本仓库承担两类功能：一是记录检索、筛选、数据整理与字段说明等项目过程；二是为后续的计量分析、可视化绘图与论文写作提供可复现的数据与文档基础。

---

## 2. 课程背景与项目制要求
根据课程课件，本课程采用“项目制·可复现”的工作流，要求学生固定数据版本与检索式、清晰组织仓库目录、完整记录字段说明与参数选择，并提供可追溯的一键运行命令或等价流程说明。课程里程碑中，M1 要求先打通“数据—检索—筛选”链条，M2 侧重计量分析产出，M3 侧重终稿与可复现 Release。课件还明确强调：参数不是细节，数据源混用却不说明偏差来源、检索口径前后不一致、阈值与参数没有记录，都会削弱方法的可解释性与可复现性。  
因此，本项目将 README、`query.yaml`、`query_rationale.md`、`query_changelog.md`、`field_dictionary.md` 等文件视为正式方法材料的一部分。

## 3. 研究问题
本项目拟重点回答以下问题：

1. 2015–2025 年间，基于 LSTM 的电力负荷预测研究发文趋势如何变化？
2. 该领域的主要研究热点、主题簇和方法演化路径是什么？
3. 哪些作者、机构和期刊在该领域具有较高活跃度或影响力？
4. LSTM、BiLSTM、Attention-LSTM 及其他混合模型在不同应用场景中的使用特征有何差异？
5. 当前研究有哪些共性不足，未来可能的发展方向是什么？

## 4. 当前数据源与检索状态

### 4.1 当前已纳入的数据源
当前项目已形成两套实际检索结果文件：
- 中国知网（CNKI）检索结果表
- Web of Science（WoS）检索结果表

CNKI 检索结果以中文文献为主，但也混入了部分“外文期刊”记录；其记录字段已稳定包含 `SrcDatabase-来源库`、`Title-题名`、`Author-作者`、`Organ-单位`、`Source-文献来源`、`Summary-摘要`、`PubTime-发表时间`、`Year-年`，部分文献还带有 `Fund-基金` 和 `DOI-DOI`。  
WoS 检索结果则是标准标签式书目格式，典型字段包括 `PT`、`AU`、`AF`、`TI`、`SO`、`DT`、`AB`、`C1`、`C3`、`RP`、`EM`、`FU`、`CR`、`TC`、`PY`、`DI`、`UT`、`DA` 等。 

### 4.2 当前数据状态判断
从当前两套检索表看，项目已完成“检索式工程化 → 检索结果导出 → 字段说明建立”的基础链条，但尚未完成严格筛选。当前结果中已有较多高度相关文献，例如 CNKI 中出现了“基于数字孪生和 TCN-BiLSTM 的电力系统短期负荷预测研究”“基于 ADKDE-LSTM 的变电站短期负荷功率区间预测研究”等直接命中文献；WoS 中也出现了含有 load forecasting、BiLSTM、attention mechanism 等关键词的英文记录。  
但与此同时，当前结果中仍混有边界较宽甚至偏题的文献，例如需求响应、综合能源系统、建筑负荷、医院能源系统、P2P 能源交易、智能决策优化等题材。也就是说，**当前检索结果是“可用的原始检索池”，不是最终分析样本”。**

### 4.3 数据源使用原则
按照课程要求，不同数据库不能在不说明偏差来源的前提下直接混合使用。  
因此，本项目当前采取如下原则：
- 保留 CNKI 与 WoS 的原始检索结果文件；
- 先分别清洗、分别记录字段说明；
- 再通过统一字段映射与筛选规则，形成后续可合并的分析数据集。

## 5. 检索策略说明

### 5.1 核心设计逻辑
当前检索式遵循统一的四层结构：

**方法词 AND 任务词 AND 场景词 NOT 排除词**

其中：
- 方法词：LSTM、长短期记忆网络、长短时记忆网络、BiLSTM、bidirectional LSTM、attention LSTM 等
- 任务词：电力负荷预测、负荷预测、短期负荷预测、电力需求预测、load forecasting、STLF 等
- 场景词：电力系统、智能电网、电网、配电网、power system、smart grid 等
- 排除词：交通流预测、网络流量预测、服务器负载预测、CPU 负载预测、bridge load prediction 等

这套口径已经写入 `query.yaml`，并且明确了中英文方法词、任务词、场景词与排除词的分组方式，以及数据库目标包括 CNKI、WanFang、VIP、Web of Science Core Collection 和 Scopus。 

### 5.2 当前检索式的工程化状态
项目最初的 V0 检索式是面向“作业展示与口径记录”的混合版表达；后续又根据数据库差异，分别向 CNKI 专业检索格式和 WoS 风格检索格式做了适配。  
当前 README 不重复粘贴全部数据库专用语法，而把具体检索式管理交给：
- `config/query.yaml`
- `reports/query_rationale.md`
- `reports/query_changelog.md`

这样做的目的，是把“研究口径”与“数据库具体语法”区分开：  
研究口径在 `query.yaml` 中统一表达；具体执行时再根据数据库差异进行语法转换。

### 5.3 当前检索的主要局限
当前检索式仍有三个明显局限：
1. “负荷预测”与“调度优化、需求响应、建筑能耗预测、综合能源系统”之间边界较近，容易产生混入；
2. CNKI 中因为 `TKA` 或篇关摘类字段覆盖面较宽，会检出部分“带负荷预测模块但主题不以此为核心”的论文；
3. WoS 的主题词检索会覆盖到楼宇、电热综合能源、医院能源管理、P2P 家庭能源交易等相邻方向，因此后续筛选必须明确纳入和排除标准。

## 6. 字段体系与 `field_dictionary.md`

### 6.1 为什么要单独建立字段说明
课程要求“字段说明完整”，这是可复现数据的重要组成部分。不同数据库的字段命名、粒度和可用性不同，如果不先建立字段说明，后续就无法可靠完成去重、筛选、计量分析和绘图。

### 6.2 当前字段体系概览
基于当前两套检索表，项目已经形成“统一字段映射 + 数据源分项说明”的字段说明思路。

#### CNKI 当前稳定字段
- `SrcDatabase-来源库`
- `Title-题名`
- `Author-作者`
- `Organ-单位`
- `Source-文献来源`
- `Summary-摘要`
- `PubTime-发表时间`
- `Year-年`
- 部分记录含 `Fund-基金`
- 部分记录含 `DOI-DOI`

#### WoS 当前典型字段
- `PT`：出版类型
- `AU`：作者缩写
- `AF`：作者全名
- `TI`：题名
- `SO`：来源期刊
- `DT`：文献类型
- `AB`：摘要
- `C1` / `C3`：机构信息
- `RP`：通讯作者
- `EM`：邮箱
- `FU` / `FX`：基金信息
- `CR`：参考文献
- `TC` / `Z9`：被引信息
- `PY`：年份
- `DI`：DOI
- `UT`：唯一标识
- `DA`：导出日期

### 6.3 统一字段映射
为便于后续合并分析，项目建议将两套数据最终映射为以下统一字段：
- `database`
- `source_type`
- `title`
- `authors`
- `authors_full`
- `affiliations`
- `source`
- `document_type`
- `abstract`
- `keywords`
- `year`
- `pub_time`
- `fund`
- `doi`
- `references`
- `cited_times`
- `unique_id`
- `subject_category`
- `corresponding_author`
- `email`

详细解释、字段用途、保留建议和清洗提示见：
- `field_dictionary.md`
